# Contributing to SAIR-Insights

Thank you for your interest in contributing! This repository is a community-driven analysis project.

## How to Contribute

1. **Fork** this repository
2. Create a **feature branch** (`git checkout -b feature/your-contribution`)
3. **Commit** your changes with clear messages
4. **Push** to your fork (`git push origin feature/your-contribution`)
5. Open a **Pull Request**

## Contribution Guidelines

- **Corrections**: If you spot factual errors in the analysis, please open an issue or PR with a clear description and supporting evidence
- **New analysis**: If you've conducted independent analysis of the SAIR competition and would like it included, please submit it as a new markdown file in the `/analysis` directory
- **Translations**: Translations of the report into other languages are welcome — please maintain the original structure and cite sources
- **Data**: Additional datasets, visualizations, or complementary data are welcome in the `/data` directory

## Code of Conduct

- Be respectful and constructive
- Cite sources for all factual claims
- Clearly distinguish between factual observations and personal opinions
- Do not share private or sensitive information from the SAIR Zulip community

## Reporting Issues

If you find errors or have suggestions, please [open an issue](https://github.com/feishu-research/SAIR-Insights/issues) with:
- A clear description of the issue
- The specific section/chapter affected
- Any supporting evidence or references
